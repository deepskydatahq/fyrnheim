# Product Epic Breakdown

Use the `product-epic-breakdown` skill for epic `$1`.

Create or update `product/stories/$1-S*.toml` files. Stories are the implementation tasks. Include `status`, `triage`, dependencies, relevant paths, and executable acceptance criteria.

Report created stories, implementation order, parallelizable groups, and any stories needing human review.
