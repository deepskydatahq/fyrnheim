# Product Values

Review and update `VALUES.md`, the ordered progression of value Fyrnheim delivers.

## Instructions

1. Read `VISION.md`, `VALUES.md`, and recent/product mission status.
2. Check whether completed missions advance any value levels.
3. Identify stale statuses or missing mission references.
4. Update `VALUES.md` only when real work or learning justifies a change.
5. Report changes and recommended next value level.

Do not treat the value ladder as a backlog or timeline. It is a prioritization heuristic: build the lowest unshipped value level first unless there is a deliberate reason not to.
