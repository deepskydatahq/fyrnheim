# Product Vision

Create or refine `VISION.md`, the north-star transformation Fyrnheim enables.

## Instructions

1. Read existing `VISION.md` if present.
2. Read `VALUES.md` for how the transformation is currently decomposed into value levels.
3. Ask clarifying questions before changing the vision if the desired transformation is ambiguous.
4. Focus on the user transformation, not features.
5. Update `VISION.md` only for fundamental pivots or major learning.
6. Report what changed and why.

The vision should answer: "What transformation are we enabling?"
